Project Overview

Why I Built This

Building an RTOS from scratch is the ultimate test of embedded systems knowledge. I wanted to deeply understand how operating systems work at the bare-metal level—from context switching to interrupt handling—rather than just using existing RTOS frameworks.

This project pushes the boundaries of what's possible on ARM Cortex-M microcontrollers, implementing features typically found in larger operating systems while maintaining the real-time guarantees critical for embedded applications.

The challenge was brutal from day one. Bare-metal ARM development means there's nothing beneath you—no libraries, no frameworks, no safety net. Just you, the processor, and a 1,000+ page ARM architecture manual. I started by hand-coding the boot sequence in assembly, setting up the stack pointer, initializing the vector table, and jumping into C. Sounds simple, but one wrong instruction and the chip just... sits there. Silent. No error messages. No debugger output. Just dead silicon.

Context switching was the real beast. I spent days poring over ARM documentation, understanding how to save and restore register states, manipulate the Program Status Register, and handle different execution modes. Every instruction matters when you're aiming for sub-microsecond switches. I wrote, tested, and rewrote the assembly code dozens of times, using a logic analyzer to measure actual timing. When I finally got preemptive multitasking working—watching multiple tasks seamlessly share the CPU with deterministic timing—it felt like magic. But it wasn't magic. It was understanding the architecture at a level most developers never need to reach.

Technical Challenges Solved

The bare-metal environment presents unique challenges. Every single operation must be carefully crafted:

  • Assembly-Level Programming: Hand-coded ARM assembly for context switching, interrupt vectors, and stack management—no libraries, no shortcuts
  • ARM Architecture Mastery: Manually worked with ARM Cortex-M registers, exception handling, and memory protection units using official ARM documentation
  • Deterministic Timing: Achieved sub-microsecond context switches through careful instruction-level optimization and understanding of CPU pipeline behavior
  • Memory Management: Created custom allocator with predictable timing—no dynamic memory fragmentation in real-time systems
  • Priority Inversion: Implemented priority inheritance protocol to prevent high-priority tasks from being blocked by low-priority ones

Core Features

  • Preemptive Multitasking: Priority-based scheduler with sub-microsecond context switching for deterministic real-time performance
  • ARM Native: Hand-optimized assembly routines leveraging ARM Cortex-M architecture features for maximum efficiency
  • Lightweight Kernel: Minimal footprint <8KB allowing use in resource-constrained embedded systems
  • Task Management: Dynamic task creation, deletion, and priority management with efficient scheduler algorithms
  • Synchronization Primitives: Semaphores, mutexes, and event flags for thread-safe communication
  • Interrupt Handling: Fast ISR processing with minimal jitter and deterministic response times
  • Memory Management: Custom allocator designed for real-time constraints with predictable allocation times
