Why I Built This

I wanted to leverage open-source ESP-NOW to create a mesh networking solution that integrates seamlessly with existing systems. The goal was simple: make device-to-device communication as easy as possible while maintaining enterprise-level security.

Most mesh solutions are either too complex to integrate or lack proper security. FPR bridges this gap by providing encryption and authentication on top of ESP-NOW's efficient protocol, making it safe for commercial deployments while remaining accessible to hobbyists.

The journey started with a problem: I was working on multiple IoT projects that needed devices to talk to each other without relying on WiFi infrastructure. ESP-NOW was perfect—fast, efficient, low-power—but it lacked the security features needed for real deployments. I could have used a proprietary solution, but that felt wrong. The community deserves open tools that are both powerful and secure.

So I dove deep into ESP-NOW's internals, studying how devices discover each other, how messages are routed, and where the security gaps were. The breakthrough came when I realized I could layer a secure handshake protocol on top without sacrificing performance. It took weeks of experimentation—balancing encryption overhead against latency requirements, designing packet formats that wouldn't fragment, implementing peer management that scales gracefully. But seeing 150+ developers clone the repository every week? That validation makes every late night worth it.

Technical Challenges Solved

The biggest challenge was implementing enterprise-grade security without sacrificing the simplicity and performance that makes ESP-NOW attractive:

  • Encryption Layer: Designed secure handshake protocol on top of ESP-NOW's MAC layer while maintaining sub-100ms latency
  • Graceful Degradation: Built comprehensive failure handling so network remains operational even when nodes disconnect or packets are lost
  • Backward Compatibility: Ensured FPR works with existing ESP-NOW systems through careful protocol design
  • Multi-Hop Routing: Implemented intelligent message routing across multiple devices without flooding the network
